# 🌤 Челябинск Погода Telegram Bot

**Полнофункциональный Telegram бот для прогноза погоды в Челябинске**

[![Deploy on Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com)

## ✨ Возможности

- 🌤 **Прогноз на 7 дней** для всех районов Челябинска
- 📊 **Почасовой детальный прогноз** (температура, осадки, ветер)
- 💡 **Автоматические рекомендации** (брать зонт, одеваться по погоде)
- 🗺 **Информация о районах** и источниках данных
- 🧭 **Полная навигация** (Назад/Главное меню/Выход)
- ⚡ **Кэширование данных** для быстрой работы

## 🚀 Быстрый старт

### Локальный запуск

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/YOUR_USERNAME/weather-chelvabinsk-bot.git
cd weather-chelvabinsk-bot

# 2. Создайте виртуальное окружение
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 3. Установите зависимости
pip install -r requirements.txt

# 4. Создайте .env файл с токеном
echo "BOT_TOKEN=ваш_токен_от_BotFather" > .env

# 5. Запустите бота
python bot.py
```

### Деплой на Render (бесплатно)

1. **Создайте бота** у [@BotFather](https://t.me/botfather)
1. **Загрузите код** на GitHub
1. **Нажмите** "Deploy on Render" выше
1. **Добавьте** переменную `BOT_TOKEN`
1. **Готово!** Бот запустится через 2 минуты

## 📋 Команды бота

| Команда | Описание |
|---|---|
| `/start` | Начать работу, показать приветствие |
| `/help` | Показать справку |
| `/menu` | Показать главное меню |
| `/cancel` | Отменить текущее действие |

## 🛠 Технологии

- **Python 3.11** - язык программирования
- **aiogram 3.x** - фреймворк для Telegram Bot API
- **aiohttp** - асинхронные HTTP запросы
- **Open-Meteo API** - бесплатный источник погоды
- **Render** - бесплатный хостинг

## 🔧 Настройка окружения

Создайте файл `.env`:

```
BOT_TOKEN=1234567890:AAxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
PORT=8080
LOG_LEVEL=INFO
```

## 📊 Мониторинг

Render предоставляет:

- **Логи** в реальном времени
- **Health check** на `/health`
- **Метрики** использования

## ❗ Важно

- Бесплатный таймер Render **засыпает через 15 минут** без активности
- Для 24/7 работы используйте **UptimeRobot** для пинга `/health` каждые 5 минут
- Данные кэшируются на **30 минут** для снижения нагрузки на API
